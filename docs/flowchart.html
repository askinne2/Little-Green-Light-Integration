üîê Data Stored in Each System

WordPress: User accounts, roles (ui_member, ui_patron_owner), membership metadata, event registrations (JetEngine CCT)
WooCommerce: Orders, products, subscriptions, payment history
Little Green Light: Constituent records, payment/gift records, membership information, event registrations, class registrations (from CourseStorm)
CourseStorm: Class catalog, student enrollments, class payments (syncs to LGL)

‚ö° Key Integration Points

WooCommerce ‚Üí Integrate-LGL: Order completion triggers automatic processing. Plugin routes orders by product category (memberships, events) to appropriate handlers. This is the primary method for new user registration and membership purchases.

JetFormBuilder ‚Üí Integrate-LGL: Form submissions trigger custom actions (lgl_renew_membership, lgl_edit_user, lgl_add_family_member, lgl_add_event_registration, lgl_deactivate_membership) that sync data to LGL. Note: User registration and membership updates via forms are deprecated in v2.0 - use WooCommerce checkout instead.

Integrate-LGL ‚Üí LGL: All WordPress/WooCommerce data flows to LGL via API. Creates constituents, records payments, updates membership status.

CourseStorm ‚Üí LGL: Direct integration (outside WordPress). CourseStorm reads membership status from LGL and writes class registrations back to LGL.
